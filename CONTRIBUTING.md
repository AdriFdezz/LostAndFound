## Before You Start

### Read the Documentation

Before making any changes, please read the [README](./README.md) of the project. It contains essential information about the structure, functionality, and setup of the application. This will help you understand the project better and avoid unnecessary changes.

----

### Check for Existing Issues and Features

If you're looking to contribute by solving bugs, please visit the [Issues](https://github.com/AdriFdezz/LostAndFound/issues) section of the project. You'll find a list of reported issues and feature requests. You can help by working on an existing issue or suggest a new one.

----

## Reporting Bugs

If you encounter any bugs while using **Lost and Found**, or if you have suggestions for improvements, we encourage you to report them so we can address them.

To report a bug, please open an issue on the [Bug report page](https://github.com/AdriFdezz/LostAndFound/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5BBUG%5D) and provide as much detail as possible, including:

- **Description of the bug**: What was the issue you encountered?
- **Steps to Reproduce**: How can we reproduce the bug?
- **Expected behavior**: What did you expect to happen?
- **Screenshots**: If possible, include screenshots to help us understand the issue.
- **Data about where did you encounter the bug**: OS, Browser, versions...
- **Additional context:** Add any other context about the problem here.

----

### Reporting a New Feature

If you have an idea for a new feature, you can also suggest it by opening an issue in the [Request New Feature page](https://github.com/AdriFdezz/LostAndFound/issues/new?assignees=&labels=enhancement&template=feature_request.md&title=%5BFEATURE%5D). Make sure to clearly describe the feature and how it would benefit the platform.

## How to Contribute

1. **Fork the Repository**: First, fork the repository to your own GitHub account and clone it to your local machine.

    ```bash
    git clone https://github.com/your-username/LostAndFound.git
    ```

2. **Create a New Branch**: It's always a good idea to create a new branch for your changes. You can create a new branch using:

    ```bash
    git checkout -b feature/your-feature-name
    ```

3. **Make Your Changes**: Make sure to follow the coding style and conventions used in the project. You can refer to the existing code for guidance.

4. **Test Your Changes**: If you added new features or fixed bugs, make sure to test them to ensure everything is working as expected.

5. **Commit Your Changes**: Once you've made your changes and tested them, commit them with a meaningful message describing what you did.

    ```bash
    git add .
    git commit -m "Description of your changes"
    ```

6. **Push Your Changes**: Push your branch to your forked repository.

    ```bash
    git push origin feature/your-feature-name
    ```

7. **Create a Pull Request**: Open a pull request (PR) on the `main` branch of the original repository. Describe the changes you made, why they're needed, and any relevant details.

    - Make sure your PR is clear and concise.
    - Include screenshots or logs if applicable.
